# Story 3.2 e 3.3: Agente Gemini, Fallback Resumes e SVG Generativo

## 1. Contexto e Objetivo
Com a Extração em GraphQL pronta, existiam dois gargalos: Os repositórios sem foto de capa (ficavam cinzas com 'default-project.png') e as descrições dos projetos careciam do toque de RH focado em C-Levels ou engenharia. Precisávamos integrar Modelos LLMs na pipeline diária e gerar código vivo pra converter bytes em arte vetorial de tipografia.

## 2. Decisões Técnicas e Trocas de Modelos
- A ideia começou usando o modelo `gemini-2.5-flash` para fazer a curadoria dos READMEs puxados do GitHub. E ele orquestraria a arte das capas (via código XML/SVG gerado).
- O SVG de Capa utiliza a propriedade fluída e a matemática `Math.floor(750 / (title.length * 0.6))` injetada no System Instruction do LLM para forçá-lo a imprimir propriedades tipográficas e `viewBox` enquadradas, previnindo que nomes gigantes de projetos vazassem da box sem usar CSS complexo no painel HTML. 

## 3. O Inferno do Rate Limit e A Solução: Exponential Backoff
Devido à pesada taxativa do serviço Free-tier Cloud da Google e a restrição limitante de **15 RPM** (Requerimentos por Minuto), a pipeline falhava sistematicamente. 
O Github Actions exibia o erro **503 High Demand** e **429 Quota Exhausted**, pois a automação tentava gerar O Sumário Geral + 4 Imagens Vetoriais SVG simultaneamente. Nessas quebras, ativava-se o nosso plano de Rollback (onde ele ignorava o Gemini e preenchia o arquivo da Vercel com falhas brancas).

**Como a Engenharia resolveu isso para Agentes Replicarem Futuramente:**
1. **Fallback Loader no Node**: Inicializamos e programamos manualmente a habilidade de carregar `readEnv` via Node nativo. Se testássemos local com `node scripts/...` ele saberia achar as flags no `.env` físico prevenindo Action crashes falsos.
2. **Setup do Repositório GitHub**: Definiu-se arquiteturalmente que *Secrets de Ambiente* quebram a herança no Node do Action. Fica documentado sob pena que `GEMINI_API_KEY` deve existir obrigatoriamente vinculada root na aba *Repository Secrets* e testável separada.
3. **Migração para 3.1-Flash**: Passamos a forçar a requisição do serviço `gemini-3.1-flash-lite-preview` provando ser infinitamente mais livre de interrupções de congestionamento na banda gratuita.
4. **Resiliência (Fetch With Retry)**: Inserção mandante e obrigatória de uma função inter-ceptadora `fetchWithRetry` escrita em JavaScript ES Module: Em todo endpoint `googleapis`, se o JSON retornar Server ou Demand error (503/429), o container aciona um `sleep(15000ms)` explícito de barreira de backoff e então recomeça no looping sem invalidar a requisição de criação dos recursos de UX e UI.
