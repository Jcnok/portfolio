# Story 4.1: Gráfico de Radar de Habilidades Autônomo (Data-Driven)

## 1. Contexto e Objetivo
As habilidades técnicas (`assets/data/languages.json`) estavam hardcoded (definidas estaticamente em valores percentuais inventados). Para um portfólio de um profissional voltado para Dados, IA e Automação focado em conversão de recrutadores técnicos, essa representação estática minava o objetivo de atestar real maestria.
O objetivo foi substituir esse mockup de radar por um painel **vivo e 100% data-driven**. O próprio sistema deve vasculhar todo o GitHub (mais de 50+ repositórios) do candidato, pesando matematicamente linhas de códigos (languages bytes) e tags (topics).

## 2. Decisões Técnicas

- **GraphQL First**: Utilizamos a API v4 do GitHub (GraphQL) via autenticação `GITHUB_TOKEN` com escopo ampliado. A consulta itera através dos repositórios do usuário resgatando blocos massivos em um pipeline único: Top 10 Linguagens Utilizadas por Repo (+ tamanho de cada microserviço) e Uso generalizado de Tags (e.g. FastAPI, DevOps).
- **Processamento via GenAI**: Em vez de fazer uma lógica complexa pra tentar traduzir "Jupyter Notebook" + "Python" de forma seca, empacota-se a matriz de estatísticas pura no prompt primário de uma AI robusta.
- **Model Switch (Rate Limit Bypass)**: Originalmente formatado para usar `gemini-2.5-flash`, identificou-se que o modelo é constantemente banido com Erros 503 (High Demand). Migrou-se a base autônoma inteira para `gemini-3.1-flash-lite-preview` que provou ser veloz, disponível na banda gratuita, suportar imenso batch textual da query, retonando um esquema formal `JSON`.
- **Remoção de Normalização Relativa**: O frontend da aplicação (`assets/js/skills-chart.js`) mantinha um divisor em que limitava e tentava achar a % máxima. Sendo que agora a GenAI retorna exatamente `0-100`, essa matemática foi extirpada limpando o código cliente.

## 3. Fluxo de Implantação
1. Desenvolvido `scripts/generate-skills.mjs` – Módulo backend de análise massiva. 
2. Carregamento de ambientes (`.env` local autônomo sem flags do node).
3. A integração CI/CD (`update-projects.yml`) recebeu um novo Job anexado garantindo que todo dia às 6 da manhã, de ponta a ponta, o repositório compila repos novos, capta vetores de capas, sumariza portfólio e, agora, sumariza suas Habilidades (Tech Stack) atualizadas para o dia.
4. Ajuste crítico em `generate-projects.mjs` convertendo o fallback model de 2.5 pra 3.1, prevenindo o preenchimento de metadados das imagens falharem.

## 4. Resultado (Acceptance)
- Pipeline executa organicamente todos os testes.
- Gráfico da home agora reflete: `Data Analysis & Python`, `AI & LLMs`, `Cloud`, entre outros blocos estritos sob avaliação da robótica do Gemini, sem intervenção humana.
- Story aceita e deployada! ✅
