## Story
**As a** usuário do portfólio,
**I want** que o gráfico radar de habilidades atualize suas cores corretamente ao trocar entre modo claro e escuro,
**so that** a experiência visual seja consistente e sem bugs visuais.

## Acceptance Criteria
1. Substituir o `setTimeout(..., 100)` em `skills-chart.js` por um `MutationObserver` no atributo `data-theme` do `<html>`.
2. As cores do gráfico (grid, angleLines, labels, pontos, tooltips) devem ser atualizadas imediatamente ao trocar o tema.
3. Não deve haver flash ou atraso visual visível na atualização do gráfico.
4. O `MutationObserver` deve ser desconectado se o elemento `<canvas>` for removido do DOM (boa prática de cleanup).
5. A funcionalidade existente do chart (dados, radar, responsividade) não deve ser alterada.

## Dev Notes

### Contexto do Sistema
[Source: docs/brownfield-architecture.md]

- **Arquivo alvo**: `assets/js/skills-chart.js` — IIFE que gerencia o Chart.js radar chart.
- **Arquivo relacionado**: `assets/js/theme-toggle.js` — Altera `data-theme` no `<html>` via `document.documentElement.setAttribute('data-theme', newTheme)` (linha 41).
- **Ordem de carregamento**: No `index.html`, `theme-toggle.js` (linha 398) é carregado ANTES de `skills-chart.js` (linha 399). Isso garante que o listener do toggle já existe quando o chart é criado.

### Problema Atual (Race Condition)
O código atual (linhas 136-155 de `skills-chart.js`) escuta cliques no botão `#theme-toggle` e espera `100ms` com `setTimeout` antes de ler o novo valor de `data-theme`. Isso é frágil porque:
1. Em máquinas lentas, o `theme-toggle.js` pode não ter processado o clique em 100ms.
2. Em máquinas rápidas, os 100ms causam um flash visual perceptível.
3. Se o botão de tema mudar de `id` ou de mecanismo de ativação (ex: shortcut de teclado), o listener de `click` em `#theme-toggle` não capturaria a mudança.

### Solução: MutationObserver
O `MutationObserver` observa diretamente a mudança do atributo `data-theme` no `<html>`, eliminando completamente a dependência temporal e a necessidade de saber como/quando o tema é alternado.

```javascript
// Padrão da solução:
const observer = new MutationObserver(() => {
  const colors = getThemeColors();
  // Atualizar todas as propriedades do chart...
  chart.update();
});

observer.observe(document.documentElement, {
  attributes: true,
  attributeFilter: ['data-theme']
});
```

### Propriedades Incompletas no Update Atual
> ⚠️ O código atual (dentro do `setTimeout`) NÃO atualiza todas as propriedades dependentes do tema. As seguintes ficam desatualizadas após troca de tema:
> - `angleLines.color` — As linhas diagonais do radar mantêm a cor antiga.
> - `pointBorderColor` e `pointHoverBackgroundColor` — Os pontos ficam com cor de borda do tema anterior.
> - Todas as propriedades de `tooltip` (`backgroundColor`, `titleColor`, `bodyColor`, `borderColor`) — O tooltip fica com as cores do tema de inicialização.
>
> Esta story **corrige** essas propriedades faltantes além de eliminar a race condition.
