## Dev Notes
### Arquitetura RAG no Chat
```
Recrutador pergunta → API /chat
  ├─ getQueryEmbedding(pergunta) → gemini-embedding-001 → vetor[768]
  ├─ retrieveTopK(vetor, vectors.json) → cosine similarity → top 5
  ├─ buildRAGContext(top5) → bloco <contexto_verificado>
  ├─ System Prompt + Context → gemini-3.1-flash-lite → resposta
  └─ Response: { reply, sources: [{title, type, score}] }
```

### Características
- **Vector Store**: Carregado 1x na cold start da Vercel Function e cacheado em memória entre warm starts
- **Graceful Degradation**: Se `vectors.json` não existir ou o embedding falhar, o chat funciona normalmente com o prompt estático (sem RAG)
- **Threshold**: Só injeta contexto com similaridade > 30% (evita noise)
- **Anti-Hallucination**: System prompt instrui explicitamente para usar APENAS dados verificados
- **Source Attribution**: Frontend exibe tag dashed com ícones (🎓 Cert / 📂 Projeto) e score de relevância
