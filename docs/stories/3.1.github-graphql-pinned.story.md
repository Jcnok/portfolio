# Story 3.1: Integração com GitHub GraphQL API

## 1. Contexto e Objetivo
Historicamente, todo o Portfólio estava formatado e acoplado localmente nos arquivos de JSON e Javascript da aplicação, criando um alto deságil na hora de publicar novos projetos. A meta dessa história foi orquestrar os resgates autônomos que alimentam a camada frontend via GitHub. 

## 2. Decisões Técnicas
- **Abordagem de API**: Utilizou-se GraphQL (API v4) em vez da REST API (v3) devido à flexibilidade de extrair a árvore massiva de `pinnedItems`, linguagens, e links primários do repositório em apenas um POST, evitando dezenas de promises HTTP sequenciais.
- **Node.js Pipeline `generate-projects.mjs`**: Arquitetado estritamente usando módulos nativos ECMAScript (ESM) sobre NodeJS 24. A integração exige injeção da flag `GITHUB_TOKEN` para autenticação.
- **Roteamento de Branch Fix**: Implementamos uma melhoria via GraphQL onde o `defaultBranchRef` (geralmente `main` ou `master`) é puxado explicitamente, garantindo que o algoritmo de imagens consiga ler capas originais localizadas em pastas de diretórios independente da nomenclatura da branch mestre.

## 3. Desafios Superados
- **Extração Híbrida de Imagens**: O código foi treinado com múltiplos testes de expressões regulares (`Regex`) de alta robustez para conseguir captar imagens tanto via tags Markdown nativas `![alt](url)` quanto via tags mistas de HTML `<img src="">`.
- **Filtros de Badges (Falsos Positivos)**: Durante testes, linhas que ilustravam os README.md como divisórias do perfil (do repositório `andreasbm/readme/assets/lines/rainbow.png`) e badges em markdown (`shields.io`, `badgen`) estavam sendo classificadas e puxadas como Imagens de Capa. Adicionou-se uma filtragem negativa rigorosa por string para bloquear esses domínios em `scripts/generate-projects.mjs`.

## 4. Resultado Final
O arquivo `assets/data/projects.json` passou a nascer automaticamente e é reescrito lendo diretamente a infraestrutura pinada, abstraindo o backend nativo da plataforma Vercel e repassando o trabalho autônomo ao CI Github Actions.
