## Story
**As a** equipe de desenvolvimento,
**I want** que os testes sejam executados automaticamente em cada push via GitHub Actions,
**so that** o deploy na Vercel seja precedido de validação de qualidade.

## Acceptance Criteria
1. Criar `.github/workflows/ci.yml`. ✅
2. O workflow deve executar `npm install` e `npm test` em cada push para `main` e em Pull Requests. ✅
3. O workflow deve falhar se algum teste falhar, impedindo merge. ✅
4. O runtime a ser usado é Node.js 20.x. ✅

## Dev Notes

### Workflow Existente
Existe um `deploy.yml` (GitHub Pages deploy) que está **deprecado** (commit `d601eee`). O `ci.yml` convive junto sem conflitos.

### Decisões Técnicas
- **`npm ci`** em vez de `npm install`: mais rápido e determinístico em CI (usa `package-lock.json` exato)
- **`cache: 'npm'`** no `setup-node@v4`: cache automático do npm para builds mais rápidos
- **Node 20.x**: conforme AC4 e compatibilidade LTS
- **actions v4**: versões mais recentes de `checkout` e `setup-node`

### Proteção de Branch (Opcional)
Para que o workflow de fato impeça merge em PRs com testes falhando, é necessário configurar **Branch Protection Rules** no GitHub:
- Settings → Branches → Add rule para `main`
- Marcar "Require status checks to pass before merging"
- Selecionar o check "Run Tests"
